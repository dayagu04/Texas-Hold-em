# PRD · 多人扑克平台

> 版本 v1.0 · 2026-06-18 · Owner: PM

## 1. 一句话定位

为**指定小圈子**（家人 / 朋友 / 同事局）打造的网页扑克房，一个站点同时支持**德州扑克、掼蛋、炸金花**，UI 精致，必要时可一键加入 AI 对手凑齐人数。

## 2. 目标与非目标

### 目标
- **三玩法统一入口**：同一登录、同一大厅、同一聊天，按桌选玩法。
- **白名单准入**：无密码登录，仅放行 `allowed_users.json` 中的用户名（沿用现状）。
- **精美 UI**：赌场绿桌 / 暗金质感 + 微动效（发牌、筹码飞入、胜负高亮）。
- **可选人机**：房主在创建或运行中可"加入 AI"补位 / 凑齐，AI 与真人共桌。
- **稳定的实时同步**：Socket.IO 推送，断线重连后能恢复到当前对局视图。

### 非目标（v1 不做）
- 真钱、充值、提现。
- 跨房间排行榜、赛事、锦标赛。
- 移动端原生 App（响应式 Web 即可）。
- 牌局复盘、AI 教学。
- 国际化（默认简体中文，文案集中在前端字典里以便日后扩展）。

## 3. 用户与场景

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| **房主** | 发起一桌局的人 | 选玩法 / 设置盲注或倍率 / 邀请人 / 召唤 AI |
| **玩家** | 受邀进入的白名单用户 | 选座、操作、聊天、看清局势 |
| **观战者** | 无座位但已进房 | 只看不动，不影响节奏 |
| **管理员** | 维护白名单的人 | 改 `allowed_users.json` 即可，不进 UI |

典型场景：周五晚上 4 个朋友想打掼蛋，但只到了 3 个 → 房主点"加 1 个简单 AI"立即开局。

## 4. 功能范围（v1）

### 4.1 账号与权限
- 登录页输入用户名 → 后端比对白名单 → 颁发 session token（HTTP-only Cookie 或 localStorage 二选一，详见 ARCHITECTURE）。
- 同一用户名同一时刻**仅允许一个活跃连接**，新连接顶替旧连接（旧端收到 `kicked`）。

### 4.2 大厅
- 房间列表：玩法 tag、桌名、当前人数 / 上限、是否含 AI、状态（等待 / 进行中）。
- 新建房间表单：玩法、桌名、初始筹码 / 倍率、座位数、是否允许观战、AI 预设（无 / 1 简单 / 1 进阶 / 自定义）。
- 加入：点击空座位入座，或点"观战"。

### 4.3 三种玩法
规则细节见 [GAME-RULES.md](./GAME-RULES.md)，本节仅列产品差异：

| 玩法 | 人数 | 一局节奏 | 关键交互 |
|------|------|----------|----------|
| 德州扑克 Texas Hold'em | 2–6 | 4 街，每街下注 | Fold / Check / Call / Raise / All-in |
| 掼蛋 Guandan | 4（2v2 固定搭档） | 出牌阶段 | 出牌 / 过牌 / 提示 / 进贡还贡 |
| 炸金花 Three-card Brag | 2–6 | 看牌或闷牌后下注 | 看牌 / 跟注 / 加注 / 比牌 / 弃牌 |

### 4.4 人机（AI Bot）
- **入口**：房主在房内任意空位点"+"，弹出"邀请玩家 / 加入 AI"二选一。
- **难度**：`easy` / `normal`（v1 两档够用，留 `hard` 接口位）。
- **行为**：与真人同步收事件、同样的 timeout（默认 25s 思考），但 AI 决策延迟 1.5–4s 以避免"机器感"。
- **可见性**：座位卡上显示 🤖 图标 + 难度徽标。

### 4.5 聊天
- 房间内文字聊天，支持系统广播（"Alice 加注 100"由前端基于事件本地渲染，不走聊天通道）。
- 全局过滤：纯前端基于敏感词列表（v1 用一个 20 词左右的简单数组，可空）。

### 4.6 断线重连
- 30s 内重连：自动恢复到原座位与手牌视图。
- 超时：本局视为 `auto-fold`（德扑/炸金花）或 `auto-pass`（掼蛋），下局保留座位但 `sitting_out=true` 直到玩家手动复位。

## 5. 非功能需求

- **延迟**：动作下发到全部客户端 < 200ms（同机房）。
- **并发**：v1 目标 5 桌 × 6 人 = 30 个并发玩家，无需水平扩展。
- **可用性**：单节点部署，重启时序：前端不可用 < 10s，房间状态可丢（v1 不做持久化）。
- **隐私**：仅自己看到自己的底牌，服务端永不把对手底牌推给你直到摊牌。
- **审计**：服务端日志记录 `(table_id, hand_id, player, action, timestamp)`，不记录手牌内容，便于复盘但不泄密。

## 6. 里程碑

| M | 内容 | 交付 |
|---|------|------|
| **M0 当前** | 德扑 MVP 已存在 | ✅ |
| **M1** | 引入 GameEngine 抽象 + 大厅多玩法切换 + Bot 抽象 | 后端 |
| **M2** | 炸金花完整玩法 + 前端炸金花桌面 | 前后端 |
| **M3** | 掼蛋完整玩法 + 前端掼蛋桌面 | 前后端 |
| **M4** | UI 精修（动效 / 音效 / 暗金主题） + 断线重连 | 前端 |
| **M5** | Bot 难度调优 + 端到端联调 + 部署文档 | 全员 |

> 前后端两个 agent 可在 M1 起并行：后端按引擎/玩法推进，前端按"骨架 + 主题 + 三个桌面组件"推进，节点在 [API-CONTRACT.md](./API-CONTRACT.md) 集合。

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 三玩法状态机差异大，引擎抽象漏抽 | 先按 [API-CONTRACT.md](./API-CONTRACT.md) 的 `state` schema 反推接口，再写引擎 |
| 掼蛋牌型判定复杂（炸弹/同花顺/火箭） | 单独建 [backend/app/game/guandan/](../backend/app/game/guandan/) 模块，配 ≥30 条单测 |
| Bot 同步阻塞主循环 | Bot 决策跑在 `asyncio.create_task` 里，不卡其他玩家 |
| 前端三种桌面代码重复 | 抽 `<TableShell>` 容器 + 三个 `<Board*>` 子组件，共享 HUD/聊天/Bot 面板 |
