# UI 设计规范

> 受众：前端 agent。所有视觉决策以本文为准；与 [API-CONTRACT.md](./API-CONTRACT.md) 联合阅读。

## 1. 设计语言

**关键词**：暗夜赌场、暗金奢华、聚焦、克制动效。

- **基调**：深绿 + 黑棕 + 暗金，避免亮色 / 卡通。
- **节奏**：玩家关注点永远是中央桌面 → 信息密度从中心向外递减。
- **动效原则**：发牌 / 翻牌 / 筹码飞入 / 胜负光晕用动画；状态切换、面板进入用 200ms 内淡入；不要使用弹跳与抖动。

### 6.1 创建房间弹窗
- 第 1 步：选玩法（三大卡片，点选高亮）。
- 第 2 步：参数（座位 / 盲注或底注 / 初始筹码）。
- 第 3 步：AI 配置（每个空座位单独一行：`座位 N · ⓘ ○ 不加 ◉ 简单 ○ 进阶`）。

## 7. 牌桌页（共通）

```
┌──── 顶部栏（房间名 / 当前手数 / 退出） ─────────────────┐
│                                                       │
│  ┌─ 玩家列表 ─┐    ┌──── 桌面（felt） ──────┐  ┌─聊天─┐│
│  │ Alice 🪙   │   │                         │  │      ││
│  │ Bob   🪙   │   │      [board 区]         │  │      ││
│  │ 🤖easy 🪙  │   │                         │  │      ││
│  │ ...        │   │                         │  │      ││
│  └────────────┘   └────────────────────────┘  └──────┘│
│                                                       │
│  ┌──────── 行动条（自己回合显示） ────────────────────┐ │
│  │ 倒计时 25s ⏱   [Fold] [Check] [Call 40] [Raise…] │ │
│  └─────────────────────────────────────────────────┘ │
└───────────────────────────────────────────────────────┘
```

- 自己始终在屏幕**底部中央**位置（其它玩家围绕桌沿分布）。
- 行动条仅在 `current_turn.sid == me` 时出现，淡入 120ms。
- Raise 弹出小滑块 + 数字输入 + 快捷按钮（½ pot / pot / all-in）。
- 倒计时进度条嵌在行动条左侧，颜色 5 秒内变红。

### 7.1 共用组件
- `<SeatCard>`：玩家头像（首字母圆形）、昵称、筹码、当前下注 chip 数、状态徽标（folded / all-in / 🤖level）。
- `<CardSprite>`：单张牌组件，支持 `face` `back` `dim`（暗淡，不在最优组合中）。
- `<ChipStack>`：筹码堆叠，按数量分层渲染（5/25/100/500 不同颜色），动画用 `transform: translate` + opacity。
- `<DealerButton>`：金色 `D` 圆形，挂在庄家位旁。

### 7.2 三种 Board 差异

#### TexasBoard
- 中央椭圆桌，5 张公共牌位居中；`pot` 数字正下方。
- 摊牌：胜者底牌高亮 + 金色光晕，非最优 3 张 dim。

#### GuandanBoard
- 4 个固定座位（北/西/南/东，南为自己）。
- 中央显示"上家出牌"区，左下角是己方手牌（按花色排序，可拖拽多选）。
- 顶部小标签显示当前级牌 `打 2`，搭档关系连线（座位 0↔2 一条暗金虚线）。
- 出牌按钮：`出牌(N)` 按所选张数动态变；`要不起`代替 pass。

#### BragBoard
- 圆桌，每位玩家面前 3 张牌（背面 / 已看亮一点点 / 摊牌全亮）。
- 中央池底数字 + ante / current_bet 两个小标签。
- 自己的牌片有"看牌"覆盖层，未看时全黑且印着金色"看牌"字样；点击后 3D 翻面动画 600ms。

## 8. 动效清单

| 名称 | 触发 | 时长 | 备注 |
|------|------|------|------|
| 发牌 | hand 开始 | 单张 220ms，错峰 80ms | 从牌堆位飞向座位 |
| 翻公共牌 | flop/turn/river | 翻面 360ms | y 轴 3D rotate |
| 筹码入池 | call/raise/all_in | 320ms | 沿曲线轨迹 |
| 赢家光晕 | hand_end | 1200ms | 金色径向渐变脉动 1 次 |
| 行动倒计时 | 我的回合 | 25s | 进度条线性 |
| Bot 思考 | bot 回合 | 1.5–4s | 座位卡上 `…` 闪烁 |

实现：优先 CSS transition / keyframes；复杂轨迹用 [Framer Motion](https://www.framer.com/motion/) 引入（新增依赖，前端 agent 在 package.json 中加）。

## 9. 可访问性

- 颜色对比 ≥ AA（金色文本不能放在浅色上）。
- 所有按钮支持键盘 focus 环（金色虚线 2px）。
- 实时事件用 `aria-live="polite"` 朗读关键动作（"Alice raised to 100"）。
