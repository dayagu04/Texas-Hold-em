# 玩法规则（v1 实现口径）

> 本文是后端实现与 QA 的**唯一规则口径**。任何模糊点先在此处补丁，再进代码。

## A. 通用约定

- 一副标准牌：52 张 + 2 王（仅掼蛋使用王）。
- 内部表示：`{rank: 2..14, suit: "♠♥♦♣", code: "As"}`，王额外 `rank=15/16, suit="J"`。
- A 在德州 / 炸金花中可大可小（A-2-3-4-5 视为最小顺）；掼蛋中 A 在 2-3-4-5-A 顺子里既可作大也可作小，详见 §C。

## B. 德州扑克 Texas Hold'em

> 已有 MVP，本节是**正式规格**，与现有 [backend/app/game/](../backend/app/game/) 对齐。

### 桌面参数
- 座位：2–6
- 初始筹码：1000（可在创桌时改 500 / 1000 / 2000）
- 小盲 / 大盲：默认 10 / 20，比例固定 1:2
- 最少加注：上一次加注额，开局首次加注 ≥ 大盲 × 2

### 流程
1. **Preflop**：发 2 张底牌 → 小盲位首先行动（heads-up 时大盲后说话规则按标准）。
2. **Flop**：发 3 张公共牌。
3. **Turn**：第 4 张。
4. **River**：第 5 张。
5. **Showdown**：未弃牌玩家亮牌，比 7 选 5 最优组合。

### 行动
| 动作 | 条件 | payload |
|------|------|---------|
| `fold` | 任意时刻 | `{}` |
| `check` | `current_bet == player.bet` | `{}` |
| `call` | 否则 | `{}` |
| `raise` | 必须 ≥ 上次加注额 | `{ amount: int }` |
| `all_in` | 任意时刻 | `{}` |

### 边池
- 多人 all-in 时按筹码分层建池，亮牌后由各池独立结算。
- 算法：按 all-in 额度排序后依次 `min_stack * active_count` 入池。

### 牌型与比较
沿用现有 `evaluator.py`：高牌 < 一对 < 两对 < 三条 < 顺子 < 同花 < 葫芦 < 四条 < 同花顺 < 皇家同花顺。

### 边界
- 全部对手弃牌：剩余玩家拿走主池，不需亮牌。
- 平局：等分，余数按位置（庄家左手第一位起）发放。

## C. 掼蛋 Guandan

> 4 人 2v2 固定搭档，对家配合升级。v1 实现"打 A"的简化变体（不实现"过 A"晋升机制，每局独立）。

### 桌面参数
- 座位：固定 4，对角为搭档。座位 0&2 为一队，1&3 为一队。
- 用牌：2 副完整牌（含 4 张王）= 108 张。
- 当前级别牌：v1 固定为 `2`（"打 2"）；扩展位预留 `level: 2..A`。

### 牌型（按强度由弱到强）
| 牌型 | 张数 | 例子 |
|------|------|------|
| 单张 Single | 1 | 5 |
| 对子 Pair | 2 | 7-7 |
| 三张 Trips | 3 | J-J-J |
| 三带二 FullHouse | 5 | 9-9-9-3-3 |
| 顺子 Straight | 5 | 5-6-7-8-9 |
| 三连对 TubePair | 6 | 6-6-7-7-8-8 |
| 钢板 Plate | 6 | 9-9-9-10-10-10 |
| 同花顺 StraightFlush | 5 | 同花 5-6-7-8-9，**强度高于普通炸弹** |
| 炸弹 Bomb | 4–10 张同点 | 张数越多越大 |
| 火箭 Rocket | 4 王 | 最大 |

**级别牌**（本局是 2）当对应同花色时**可作癞子**替代任意非王牌（v1 简化：仅识别为级牌，不做癞子；预留 `wildcard` 接口）。

### 比较规则
- 同型同张数：比关键点数。
- 炸弹比顺序：4 炸 < 5 炸 < ... < 8 炸 < 同花顺 < 9 炸 < 10 炸 < 火箭。（v1 采用此简化序）
- 顺子：A 可作 1（A-2-3-4-5）也可作大（10-J-Q-K-A）。
- 不同牌型不可比，**炸弹/火箭可压一切非更大炸**。

### 流程
1. **发牌**：每人 27 张。
2. **进贡 / 还贡**（v1 简化版，可在创桌时关闭）：
   - 上一局败方（或随机方，首局）头家先出。v1 首局直接由"持有红心 4"的玩家先出，关闭进贡。
3. **出牌**：从首家起逆时针出牌。每名玩家可：
   - `play`：出一组合法牌型，需大于场上当前牌（除非自己开了一轮）。
   - `pass`：放弃；连续三家 pass 则上一个出牌人**重新开张**，可出任意牌型。
4. **结算**：先出完者名次为 1（"头游"），其搭档名次决定胜负。v1 结算口径：
   - 双上：1、2 名为同队 → 大胜（+3 分）
   - 一三：1、3 名为同队 → 小胜（+2 分）
   - 一四：1、4 名为同队 → 平胜（+1 分）

### 行动
| 动作 | payload |
|------|---------|
| `play` | `{ cards: Card[] }` |
| `pass` | `{}` |
| `hint` | `{}` 客户端可发，服务端返回 `hint` 事件示意一组合法牌（可选 v1 不实现） |

### 边界
- 不允许"分开炸弹"出非炸牌型（合法性校验放 `combos.py`）。
- 王只能组成对王 / 火箭 / 不参与单张顺子。
- 出牌后立刻校验是否清手 → 若清 → 标记名次 → 检查胜负是否已定。

## D. 炸金花 Three-Card Brag（中式简化）

### 桌面参数
- 座位：2–6
- 初始筹码：1000
- 底注 ante：默认 10（创桌可改）
- 最低跟注：当前注；最高单次加注：当前注 × 2（俗称"一注顶两注"），可关。

### 牌型（由强到弱）
1. **豹子 Trips**：3 张同点（最大 AAA）
2. **同花顺 Straight Flush**
3. **同花 Flush**
4. **顺子 Straight**：A-2-3 视为最小顺，A-K-Q 最大
5. **对子 Pair**
6. **散牌 High Card**

特殊：`235` 不同花在部分玩法中可压豹子，v1 **不启用此规则**。

### 流程
- 每人发 3 张暗牌。
- 庄家位左侧首发。每位玩家可：
  - `look`：看自己的牌（首次看牌后，跟注 / 加注金额翻倍计入底池）。
  - `call`：跟当前注。
  - `raise`：在合法范围内加注。
  - `compare`：与某指定对手比牌（必须先看牌；必须支付双倍当前注），输的一方弃牌出局。
  - `fold`：弃牌。
- 终局：仅剩 1 人 → 拿走全部底池；或所有剩余玩家被强制比牌（默认连续 N 圈无加注后摊牌，N=3）。

### 行动
| 动作 | payload | 备注 |
|------|---------|------|
| `look` | `{}` | 仅自己生效；公开广播 `looked=true` 标记 |
| `call` | `{}` | 已看牌者付 2× |
| `raise` | `{ amount }` | 必须 ≥ 当前注，已看牌者付 2× |
| `compare` | `{ target_sid }` | 双方亮牌，付费方为发起方 |
| `fold` | `{}` | |

### 边界
- 一桌仅剩 2 人时，禁止 `raise`，只能 `call` / `compare` / `fold`。
- 看牌前禁用 `compare`。
- 比牌平局：发起方判负（庄家规则），与"挑战者吃亏"传统一致。
