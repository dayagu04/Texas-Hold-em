# 012 - 移动端响应式 + 音效系统

> **编号**: 012
> **状态**: 待排期
> **优先级**: P2(体验增强,非阻塞)
> **依赖**: 现有牌桌组件(TexasBoard / TableShell / SeatCard)
> **契约**: 无新接口(纯前端)
> **预估**: 前端 4-6h(响应式 3-4h + 音效 1-2h)

## 1. 需求背景

当前牌桌按桌面横屏布局设计,手机上座位环绕、操作栏、聊天都会挤变形。同时全程静音,缺少下注/发牌/轮到你的听觉反馈。两者都是 P2 体验项,不阻塞核心玩法。

## 2. 方案

### 2.1 移动端响应式牌桌

断点:`< 768px` 视为移动端(沿用 Tailwind `md:` 断点)。

**桌面(现状)**:椭圆桌,座位环绕一圈。

**移动端(窄屏)线框**:

```
┌─────────────────────────┐
│  对手横排(顶部滚动条)    │   ← 其余玩家压缩成横向小卡,可横滑
│  [P2][P3][P4][P5]        │
├─────────────────────────┤
│                          │
│      公共牌区             │   ← 居中,牌略缩小
│   [♠A][♥K][♦Q] pot:120   │
│                          │
├─────────────────────────┤
│   我的座位(底部固定)     │   ← 自己永远在底部,手牌大显示
│   [♣J][♣10]   chips:980  │
├─────────────────────────┤
│ [弃牌][跟注][加注] ⏱     │   ← 操作栏贴底,大按钮,带倒计时
└─────────────────────────┘
```

要点:
- 自己固定底部、手牌放大;其余玩家压成顶部横排小卡(超出可横滑)。
- 操作栏贴底,按钮加大(触摸友好,≥ 44px 高)。
- 聊天/日志收进可展开抽屉(底部上滑或一个图标),不常驻占地。
- 公共牌、筹码、pot 居中,牌尺寸随屏宽缩放。
- 倒计时进度环用契约新增的 `turn_total_ms` 画(见 §2.4),不再硬编码总时长。

实现建议:用 CSS 媒体查询/Tailwind 响应式类做两套布局,而非 JS 测宽切换组件,避免重渲染抖动。座位定位现在大概率是绝对定位环绕——移动端改 flex 纵向堆叠。

### 2.2 音效系统 + 全局静音

触发点清单(最小集):

| 事件 | 音效 | 来源事件 |
|------|------|---------|
| 发牌/开新局 | 洗牌/发牌声 | `table:state` stage 进入 preflop |
| 轮到自己 | 提示音(叮) | `current_turn.sid == 我` |
| 下注/加注 | 筹码声 | `table:state` log 新增下注动作 |
| 弃牌 | 翻牌声 | log 新增 fold |
| 摊牌/获胜 | 胜利音 | `table:hand_end` 我 result=won |
| 聊天 | 轻提示(可选,默认关) | `table:chat` |

- 全局静音开关:放在顶部菜单或设置,状态存 `localStorage`,默认**开启声音**(或保守起见默认静音,避免突兀——本期定**默认静音**,用户主动开)。
- 音频文件放 `frontend/public/sounds/`,用 `Audio` 或 Web Audio API 预加载。
- 移动端浏览器音频需用户首次交互后才能播放——首次点击/入座时解锁 AudioContext。

## 4. 验收标准

- [ ] iPhone/Android 主流尺寸(375/390/414 宽)牌桌不溢出、不重叠
- [ ] 自己座位固定底部,手牌清晰;对手横排可横滑
- [ ] 操作按钮触摸友好(≥ 44px),倒计时可见
- [ ] 聊天可展开/收起,不常驻挤占牌桌
- [ ] 各触发点音效正确播放;静音开关即时生效并跨刷新保持
- [ ] 桌面端布局不受影响(回归)
