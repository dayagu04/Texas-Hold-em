# 015 - 牌桌 UI 真实化重构（对标欢乐斗地主）

> **编号**: 015
> **状态**: 待排期
> **优先级**: P1（产品质感，独立 sprint）
> **依赖**: 现有 TexasBoard/GuandanBoard/BragBoard、framer-motion（已装）、`theme/tokens.css`
> **设计**: [UI-DESIGN.md](../design/UI-DESIGN.md)（本需求是其落地强化，不另立设计语言）
> **预估**: 前端 8-12h（分阶段，可拆多个 sprint）

## 1. 需求背景

M5 验收时 PM 反馈：当前牌桌 UI"不够真实，需要重构，参考腾讯欢乐斗地主"。#014 只修了聊天遮挡这一个点，**视觉沉浸度的大头没做**。

**重要现状（避免推倒重来）**：牌桌**已经有**基础——`TexasBoard.tsx` 已实现 felt 椭圆渐变、暗角 vignette、framer-motion 发牌/翻牌动画、金色光晕，framer-motion 已装，`tokens.css` 设计令牌齐全，[UI-DESIGN.md](../design/UI-DESIGN.md) 也已定义了完整动效清单和视觉规范。

**问题不是"没有"，是"不够成品级"**：现有实现停在"能看"，离欢乐斗地主那种"质感拉满、反馈到位、一眼高级"还有距离。本需求是把已有基础**提升到成品保真度**，不是从零重写。

## 2. 对标分析：欢乐斗地主为什么"真实"

拆解腾讯欢乐斗地主的视觉/交互要素，对照本项目差距：

| 要素 | 欢乐斗地主 | 本项目现状 | 差距 |
|------|-----------|-----------|------|
| 桌面材质 | 厚重绒布 + 高光 + 立体边框 | 平面 radial 渐变 | 缺纹理、立体感、桌沿 |
| 卡牌 | 厚度阴影、圆角、翻牌实感 | 扁平 SVG | 缺厚度/材质/翻牌手感 |
| 筹码/积分 | 立体筹码堆、飞入动画带轨迹+音 | ChipStack 基础堆叠 | 缺立体感、入池轨迹动画 |
| 座位/头像 | 圆形头像框 + 状态光环 + 名牌底板 | SeatCard 基础卡 | 缺质感底板、当前行动高亮呼吸 |
| 行动反馈 | 出牌/下注有放大+音效+飘字 | 基础 | 缺"打出去"的力度反馈 |
| 胜负演出 | 赢家光柱 + 金币雨 + 庆祝动画 | 金色光晕脉动1次 | 缺庆祝级演出 |
| 整体氛围 | 暖光聚焦中心、周边压暗 | 已有 vignette | 基本够，可强化聚光 |

## 3. 方案（分三阶段，可独立交付）

### 阶段 A：材质与质感（P1，4-5h）— 静态先到位

不碰逻辑，只提升视觉保真度：

1. **桌面**：felt 渐变叠加绒布噪点纹理（CSS `background-blend` 或一张 tileable 噪点 PNG），加桌沿立体边框（内阴影 + 金色细描边 + 外投影），让桌子"浮起来"。
2. **卡牌（CardSprite）**：加厚度感（底部 2-3px 深色边 + 圆角 + `--shadow-card`），牌背设计统一暗金花纹。dim 态（非最优组合）做半透明下沉。
3. **座位（SeatCard）**：头像圆框加金属质感描边，名牌做底板（暗色圆角 + 金边），筹码数用等宽字体。当前行动者：金色呼吸光环（`animate` 循环 scale+opacity）。
4. **筹码（ChipStack）**：立体筹码（径向高光 + 叠层投影），按面额配色（5白/25绿/100黑/500紫，对齐 UI-DESIGN §7.1）。

### 阶段 B：动效与反馈（P1，3-4h）— 动起来

强化已有动画 + 补缺失的反馈：

1. **发牌**：从牌堆位（庄家旁）飞向各座位，错峰（已有雏形，强化轨迹+落定回弹抑制）。
2. **筹码入池**：call/raise/all-in 时筹码沿贝塞尔曲线飞入中央 pot（UI-DESIGN §8 已规划，落地）。
3. **下注/出牌反馈**：动作瞬间座位卡轻微放大 + 飘字（"加注 100"金色上浮淡出）。
4. **翻公共牌**：3D y 轴翻面（已有，确认 flop 三张错峰、turn/river 单张）。
5. **赢家演出**：金色光柱 + 可选金币雨（framer-motion 粒子或 CSS），比现在的"脉动1次"更有庆祝感。

### 阶段 C：音效联动 + 三玩法对齐（P2，2-3h）

1. 视觉动作与 #012 音效系统联动（发牌声、筹码声、翻牌声、胜利声触发点对齐动画时机）。
2. 把 A/B 的质感提升同法应用到 GuandanBoard / BragBoard（先 Texas 打样，后两者复用组件和令牌）。

## 4. 约束

- **复用现有令牌**：颜色/圆角/阴影/动效时长全部用 `tokens.css` 的 CSS 变量，不新增散装值。需要新令牌先加进 `tokens.css` 并同步 [UI-DESIGN.md](../design/UI-DESIGN.md) §2。
- **不改牌局逻辑**：纯视觉/动效层，不动 socket 时序、状态机、行动合法性。重连/摊牌/自动开局行为保持不变。
- **性能**：动效用 `transform`/`opacity`（GPU 友好），避免触发 layout。移动端降级（粒子/复杂动画可关）。
- **桌面端优先打样，移动端不回归**：#012 的移动端响应式布局不能被破坏。
- 新增美术资源（噪点纹理、牌背花纹）放 `frontend/public/`，体积控制（纹理 ≤ 50KB，用 tileable 小图）。

## 6. 验收标准

- [ ] 桌面有绒布质感 + 立体桌沿，不再是平面渐变
- [ ] 卡牌有厚度/材质，牌背统一暗金花纹，dim 态明显
- [ ] 当前行动者有呼吸光环，一眼能看出轮到谁
- [ ] 下注/加注时筹码飞入 pot + 飘字反馈
- [ ] 赢家有庆祝级演出（光柱/金币雨），不只是一次脉动
- [ ] 整体观感对标欢乐斗地主级别（PM 主观验收）
- [ ] 三玩法（德扑/掼蛋/炸金花）质感一致
- [ ] 桌面端零逻辑回归，移动端布局不破坏，动效不卡顿
