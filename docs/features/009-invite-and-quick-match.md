# 009 - 邀请分享 + 快速匹配

> **编号**: 009
> **状态**: 待排期
> **优先级**: P1(产品力/拉新留存)
> **依赖**: 房间号简短自增格式(已上线, commit ec62912);公开主页+延迟登录(#004)
> **契约**: [API-CONTRACT.md](../design/API-CONTRACT.md) §2.3(`lobby:quick_match` / `lobby:no_match`)
> **预估**: 前端 3-4h,后端 1-2h

## 1. 需求背景

当前进房只能在大厅手点列表,缺两个常见的"社交开局"路径:

1. **邀请朋友**:房主想把房间发给朋友,只能口头报房间号。没有"复制房间号/链接"的快捷动作,也没有 `/table/000123` 这种可分享深链。
2. **快速开局**:新用户进来想立刻玩,要先看懂大厅列表、判断哪个房没满。缺一个"按玩法一键匹配"的入口。

两者都基于已上线的简短房间号(`000123` 形式)和 #004 的延迟登录能力。

## 2. 方案

### 2.1 邀请分享(纯前端为主)

房间页顶部(TableShell 头部)加一行:

```
房间 #000123  [📋 复制房间号]  [🔗 复制邀请链接]
```

- **复制房间号**:`navigator.clipboard.writeText("000123")`,toast「已复制房间号」。
- **复制邀请链接**:复制 `${location.origin}/table/000123`,toast「邀请链接已复制」。
- 深链路由 `/table/:tableId`:
  - 已登录 → 直接尝试入座(走现有 `lobby:join_table`,座位自动选最小空位;满了则以观战进入,沿用现有 `spectate` 能力)。
  - 未登录 → 复用 #004 的延迟登录:先渲染牌桌只读骨架/或弹登录,登录成功后再 `join_table`。**关键**:登录弹层关闭后要带着 `tableId` 继续入座,不要丢掉来源。
- 房间不存在(`TABLE_NOT_FOUND`)→ 友好提示「房间已解散」并给「返回大厅」按钮。

### 2.2 快速匹配

主页/大厅顶部加「快速开局」按钮组(按玩法):

```
快速开局:  [德州扑克]  [掼蛋]  [炸金花]
```

点击发 `lobby:quick_match { game_type }`。后端逻辑:

1. 在 lobby 里筛 `game_type 匹配 && status == "waiting" && seats_taken < seats_total` 的房间。
2. 命中多个时选**最近活跃/最新创建**的一个(实现简单起见可选 `seats_taken` 最多但未满的,更快凑齐开局)。
3. 命中 → 自动入座最小空位,回 `lobby:joined`(沿用现有事件)。
4. 无命中 → 回 `lobby:no_match { game_type }`,前端引导「没有可加入的房间,帮你创建一个?」→ 打开创建房间弹层并预选该玩法。

> 设计选择:`quick_match` **不自动创建房间**,只匹配或引导创建。自动创建会产生大量空房,污染大厅,交给用户确认更干净。

## 3. 前后端分工

### 后端
- [ ] `lobby.py`:`quick_match(game_type) -> table_id | None` 选房逻辑
- [ ] `sio.py`:`@sio.on('lobby:quick_match')`,命中走现有 join 流程并 emit `lobby:joined`,无命中 emit `lobby:no_match`
- [ ] 复用现有入座/座位分配/广播逻辑,不重复实现
- [ ] 测试:有空房命中、满房跳过、playing 房跳过、无房回 no_match

### 前端
- [ ] 路由新增 `/table/:tableId`,挂载牌桌容器并触发入座
- [ ] 未登录深链:接 #004 延迟登录,登录后带 tableId 续入座
- [ ] TableShell 头部:复制房间号 / 复制邀请链接两个按钮 + toast
- [ ] 主页/大厅:快速开局按钮组,发 `lobby:quick_match`
- [ ] 接 `lobby:no_match` → 打开创建弹层并预选玩法
- [ ] 文案全中文(zhCN)

## 4. 验收标准

- [ ] 房间页能一键复制房间号和 `/table/000123` 链接,有 toast 反馈
- [ ] 新标签页打开邀请链接,已登录直接入座对应房间
- [ ] 未登录打开邀请链接,登录后自动进入该房间(不丢 tableId)
- [ ] 邀请链接指向已解散房间时,提示房间已解散并可返回大厅
- [ ] 点「快速开局-德州扑克」,有等待中未满的德扑房时直接入座
- [ ] 无可加入房间时,提示并引导创建(预选玩法)
- [ ] 满员/进行中的房间不会被 quick_match 选中
