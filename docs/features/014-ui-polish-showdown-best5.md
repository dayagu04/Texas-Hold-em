# 014 - UI 打磨：摊牌最佳5张 + 真实化布局

> **编号**: 014  
> **状态**: M5 验收反馈，立即修复  
> **优先级**: P1（体验）  
> **依赖**: #010 摊牌显示已实现  
> **预估**: 前端 2-3h

## 1. 需求背景

M5 验收发现两个体验问题：

1. **摊牌只显示手牌2张不合理**：德扑摊牌应显示"手牌+公牌组成的最佳5张"（如 [A♠K♠][Q♠J♠10♠]→皇家同花顺），而非只显示手牌 [A♠K♠]。当前 HandEndModal 只渲染 `results[].cards`（2张手牌），玩家看不到最终成牌。
2. **UI 布局不够真实**：聊天悬浮球遮挡当前牌型显示区；整体风格不如腾讯欢乐斗地主沉浸。需参考真实棋牌 UI 重构布局。

## 2. 方案

### 2.1 摊牌显示最佳5张（逻辑+UI）

**数据来源**：
- 后端 `hand_end.results[].cards` 只有手牌2张（契约 §2.4 未要求下发最佳5张）。
- **前端自己算**：拿到 `results[].cards`（手牌）+ `table:state` 的 `community`（公牌），用前端已有的 `evaluator` 逻辑（或简化版）组合出最佳5张 + 牌型名。

> 为何不让后端算？回放、历史对局等场景下公牌已进 `hand_end`，前端有 community 数据；引擎结束时已有 evaluator 结果，但未下发最佳5张组合——若改契约让后端下发，需回头改 #010+#013，工作量大。**本期前端自己算，快速修复。**

**实现**：
- HandEndModal 收到 `results` 后，对每个玩家：
  - 若有 `cards`（手牌），取当前 `state.payload.community`（公牌），用 `evaluateBestHand(hole, community)` 算出 `{ best5: Card[], rank: string }`。
  - 渲染时显示 **best5**（5张高亮）+ 牌型名，而非只显示2张手牌。
- 弃牌者仍标注"已弃牌"不显示牌（保持 #010 逻辑）。

### 2.2 UI 真实化重构（参考腾讯欢乐斗地主）

对标腾讯欢乐斗地主的沉浸式设计要点：

1. **聊天不遮挡核心信息**：
   - 当前聊天悬浮球固定右下，遮挡当前行动者的牌型显示。
   - 改为：聊天窗口**收起时只显示一个小图标**（右下角，不占核心区域），**展开时浮层覆盖两侧空白**（不遮牌桌中心、筹码、行动条）。
   - 移动端：聊天收进底部抽屉，上滑展开。

2. **牌桌中心突出**：
   - 公共牌区、pot、当前行动者高亮要在视觉中心，不被聊天/日志/按钮挤占。
   - 调整 z-index 层级：牌桌 > 行动条 > 聊天（浮层）。

3. **暗金赌场氛围强化**（可选，本期可只做布局不动配色）：
   - 绿毡桌面、金色筹码、柔光高光（现有 Tailwind 主题已有 `bg-elev` / `text-gold`，保持即可）。
   - 动效：筹码飞入 pot、发牌动画（本期不强求，标注为后续优化）。

**本期重点**：先修聊天遮挡问题（收起/展开状态 + 浮层位置），其余视觉打磨标 P2 后续迭代。
