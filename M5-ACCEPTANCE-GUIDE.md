## 核心验收流程（按顺序）

### ✅ 第一步：登录 + 白名单管理（#008）

1. 登录为 **admin**（第一个用户，应该是管理员）
2. 进入「个人中心」，应该看到「白名单管理」入口
3. 点击进入，添加一个新用户（如 `测试用户`）
4. 新标签页/无痕模式，用新用户登录，应该成功

**验收点**:
- [ ] admin 能看到「白名单管理」入口
- [ ] 非 admin 用户看不到该入口
- [ ] 添加的新用户立即可登录（不重启后端）

---

### ✅ 第二步：完整游戏 + 回放（#013）

1. **admin** 创建房间（德州扑克，2人，添加1个bot）
2. 点「开始游戏」
3. 随便操作几步（下注/跟注/弃牌都行）
4. 等游戏结束
5. 返回「个人中心」→「对局历史」
6. 找到刚才的对局，点击「**回放**」按钮

**验收点**:
- [ ] 回放 Modal 弹出
- [ ] 显示玩家、公共牌、pot
- [ ] 可以点「播放」逐步重建游戏过程
- [ ] 可以「上一步」/「下一步」
- [ ] 动作历史正确显示（谁 fold/call/raise）

---

### ✅ 第三步：积分榜（#011）

1. 大厅左侧应该有「积分榜」
2. 切换三个 Tab：「积分」/「净胜」/「胜率」
3. 每次切换应该重新排序

**验收点**:
- [ ] 三个维度都能正常显示
- [ ] bot 不出现在榜上
- [ ] 个人中心显示「我的排名」（或"未上榜"）

---

### ✅ 第四步：移动端布局（#012）

1. 在游戏桌内，按 **F12** 打开开发者工具
2. 点击「手机图标」（Toggle device toolbar）
3. 选择 iPhone SE 或 Pixel 5（375px 宽）
4. 观察布局变化

**验收点**:
- [ ] 自己的座位固定在底部（手牌大显示）
- [ ] 其他玩家在顶部横排（可横滑）
- [ ] 公共牌居中
- [ ] 操作按钮贴底，够大（触摸友好）

---

### ✅ 第五步：快速匹配（#009）

1. 回到大厅
2. 点「快速开局」，选择一个玩法
3. 如果有空房，应该自动进入
4. 如果无空房，应该引导创建房间

**验收点**:
- [ ] 有空房时自动入座
- [ ] 无空房时提示创建

---

### ✅ 第六步：邀请链接（#009）

1. 在游戏桌内，找到「邀请」或「复制链接」按钮
2. 复制房间链接（如 `http://localhost:5173/table/000001`）
3. 新标签页打开该链接
4. 应该直接进入该房间

**验收点**:
- [ ] 已登录用户直接入座
- [ ] 未登录用户登录后自动进入房间

## 遇到问题怎么办？

### 常见问题

**Q: 前端报错 "Failed to fetch"**  
A: 检查后端是否启动（http://localhost:8000/api/health 应该返回 `{"status":"ok"}`）

**Q: 创建房间后卡住**  
A: 检查后端终端是否有报错，刷新页面重试

**Q: 回放按钮点击无反应**  
A: 按 F12 查看 Console 是否有错误，截图发给开发

**Q: 移动端布局不变**  
A: 确认窗口宽度 < 768px，刷新页面
